## Remediation Steps

1. Block the source IP immediately at the firewall level: `sudo ufw deny from 203.0.113.45 to any && sudo ufw reload` — verify with `sudo ufw status`
2. Install and configure fail2ban for automatic SSH brute force protection: `sudo apt install fail2ban -y && sudo systemctl enable --now fail2ban && sudo fail2ban-client status sshd`
3. Disable SSH password authentication (enforce key-only): Edit `/etc/ssh/sshd_config` — set `PasswordAuthentication no` and `PermitRootLogin no`, then restart: `sudo systemctl restart sshd`
4. Implement SSH key-based authentication if not already done: `ssh-keygen -t ed25519 -C "admin@web-server-01" && ssh-copy-id -i ~/.ssh/id_ed25519.pub user@web-server-01`
5. Consider restricting SSH access to known IPs using AllowUsers or firewall rules: In `/etc/ssh/sshd_config` add `AllowUsers yourusername@YOUR_IP_ONLY`
